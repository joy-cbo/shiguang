# 拾光博客 — 小白一键部署指南

> 全程不需要写代码。跟着点鼠标就行。20 分钟搞定。

---

## 你需要什么

- 一个 GitHub 账号（https://github.com 免费注册）
- 一个 Cloudflare 账号（https://dash.cloudflare.com 免费注册）

---

## 第一步：Fork 仓库

1. 打开 https://github.com/joy-cbo/shiguang
2. 点右上角 **Fork** 按钮
3. 点 **Create fork**

仓库复制到了你自己的 GitHub 账号下。

---

## 第二步：创建 D1 数据库

> ⚠️ 这一步必须在部署之前做！不然部署会报错。

1. 打开 https://dash.cloudflare.com
2. 左侧菜单 **Workers & Pages** → **D1**
3. 点 **创建数据库**
4. 名称填 `shiguang-db` → 点创建
5. **记下数据库 ID**（一串字母数字横杠，后面绑定要用）

---

## 第三步：创建 R2 存储桶

1. 左侧菜单 **R2**
2. 点 **创建存储桶**
3. 名称填 `shiguang-files` → 点创建
4. 进入存储桶 → **设置** → **公开访问** → 点 **允许访问**

---

## 第四步：创建 KV 命名空间（可选，但推荐）

> KV 用于缓存站点配置、标签等数据，提升性能。

1. 左侧菜单 **KV**
2. 点 **创建命名空间**
3. 名称填 `shiguang-cache` → 点创建

---

## 第五步：创建 Pages 项目

1. 左侧菜单 **Workers & Pages** → 点 **创建** → **Pages**
2. 点 **连接到 Git**
3. 选 **GitHub** → 授权登录
4. 选你 Fork 的 `shiguang` 仓库 → 点 **开始设置**

---

## 第六步：配置构建设置

| 设置项 | 填什么 |
|--------|--------|
| 项目名称 | `shiguang` |
| 生产分支 | `master` |
| 构建命令 | `npm install --prefer-offline --no-audit && npm run build` |
| 输出目录 | `dist` |

> ⚠️ 构建命令开头要加 `npm install`。不要只用 `npm run build`，否则会报错。
> ⚠️ 输出目录是 `dist`，不是 `.output`。

**环境变量（必填！）** 点「添加变量」：

| 变量名 | 值 |
|--------|-----|
| `JWT_SECRET` | 随便打一串（至少 32 个字符，字母数字混合） |

> JWT_SECRET 是登录加密用的钥匙。比如 `wodebokeshiguang2026zuibang!!`  
> ⚠️ 不要用例子里的值，自己随便想一串！

填完点 **保存并部署**。

---

## 第七步：绑定数据库、存储桶和 KV

部署会先跑一次（会失败，因为还没绑定——这是正常的）。

部署跑完后：

1. 进入 Pages 项目 → 点 **设置** → **绑定**
2. 点 **添加**：
   - 类型选 **D1 数据库** → 变量名填 `shiguang-db` → 选 `shiguang-db`
3. 再点 **添加**：
   - 类型选 **R2 存储桶** → 变量名填 `shiguang-files` → 选 `shiguang-files`
4. 再点 **添加**（可选）：
   - 类型选 **KV 命名空间** → 变量名填 `CACHE` → 选 `shiguang-cache`

---

## 第八步：重新部署

绑定改完后需要重新部署才能生效：

1. 点 **部署** → **查看所有部署**
2. 找到最新的那条（失败的）→ 点右边 `⋯` → **重试部署**

等一两分钟。看到 ✅ 绿色对勾就成功了。

部署成功后访问的网址类似：
```
https://shiguang-xxxx.pages.dev
```

---

## 第九步：初始化博客

1. 浏览器打开网址，后面加上 `/admin/setup`
   - 例如：`https://shiguang-xxxx.pages.dev/admin/setup`
2. 填用户名 + 密码（密码至少 8 位）
3. 点「初始化」

**数据库自动建表。博客上线！**

---

## （可选）绑定自定义域名

1. Cloudflare 后台 → 你的域名 → DNS → 添加记录
   - 类型 CNAME，名称 `@`，目标填 `shiguang.pages.dev`
2. 回到 Pages 项目 → **自定义域** → 输入你的域名 → 保存

---

## 以后再改代码

每次修改代码推送到 GitHub：

```bash
git add -A
git commit -m "改了xxx"
git push
```

Cloudflare 看到 GitHub 有新代码，**自动重新构建部署**——什么都不用管。

## 遇到问题？

| 现象 | 解决 |
|------|------|
| 初始化页面打不开 | 确认 JWT_SECRET 环境变量有没有填 |
| 部署报错 8000022 | 确认第二步 D1 数据库有没有创建 |
| 数据库报错 | 确认第七步 D1/R2 绑定有没有做 |
| 安装依赖报错 | 确认第六步构建命令是 `npm install && npm run build` |
| 搜索不工作 | 确认 D1 数据库已绑定，系统会自动创建 FTS5 索引 |
| 缓存不生效 | 确认 KV 已绑定（可选，不影响核心功能） |
| 其他问题 | 发邮件：joy.cb@qq.com |
